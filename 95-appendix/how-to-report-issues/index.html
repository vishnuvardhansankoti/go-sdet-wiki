    How to Report Issues for This Wiki¶

    If you find mistakes, broken links, unclear explanations, or technical issues, please report them using the project's GitHub Issues page.

    • Report issues here: Go SDET Wiki Issues

    What to Report¶

    Please open an issue for:

    1. Incorrect technical information
    2. Broken or outdated links
    3. Typos or unclear wording that affects understanding
    4. Build or setup problems
    5. Code examples that fail unexpectedly
    6. Navigation or formatting problems in the site

    Before Creating an Issue¶

    1. Check whether the issue already exists in Open Issues.
    2. Confirm the problem on the latest published content (or latest repository version if applicable).
    3. Gather minimal evidence to reproduce the issue.

    How to Create a Good Issue¶

    1. Use a Clear Title¶

    Good title examples:

    • "Broken link in Install and Setup page"
    • "Example in Variables and Types fails to compile"
    • "Rancher Desktop setup section missing Linux step"

    2. Include Key Details¶

    Add the following details in the issue body:

    1. Page URL where the issue appears
    2. Section heading name
    3. What you expected to happen
    4. What actually happened
    5. Steps to reproduce
    6. Environment details (OS, browser, Go version, tools)

    3. Add Supporting Evidence¶

    When possible, include:

    1. Screenshot of the page or error
    2. Error logs or command output
    3. Relevant code snippet

    Issue Template You Can Copy¶

    ## Summary

URL: <page-url>

1. ...
2. ...
3. ...

## Environment
- OS:
- Browser:
- Go version:
- Tools/IDE:

## Evidence
Attach screenshots, logs, or snippets if available.

    Best Practices for Faster Resolution¶

    1. Keep each issue focused on one problem.
    2. Use descriptive titles instead of generic labels like "bug".
    3. Provide reproduction steps that someone else can follow exactly.
    4. If possible, suggest a fix direction.

    Security Note¶

    Do not post secrets, credentials, API keys, tokens, or private data in public issues.

    If you discover a sensitive security problem, report it responsibly and avoid publicly disclosing exploit details.

    Additional Contribution Option¶

    If you already know the fix, you can open a pull request directly after creating or referencing an issue:

    • Repository: go-sdet-wiki
    • Issues: Issue Tracker
