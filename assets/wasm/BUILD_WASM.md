# Building the Go WASM Playground Binary

## Overview

The `go.wasm` file is a compiled WebAssembly binary built from Go code. This file needs to be generated by compiling Go code with specific GOOS and GOARCH environment variables.

## Prerequisites

- Go 1.11+ installed
- `wasm_exec.js` from the Go distribution (included in this directory)

## Building go.wasm

### Create the Go Playground Program

Create a file `cmd/wasm/main.go`:

```go
package main

import (
    "fmt"
    "syscall/js"
)

func main() {
    c := make(chan struct{}, 0)
    
    // Register a function that can be called from JavaScript
    js.Global().Set("runPlayground", js.FuncOf(func(this js.Value, args []js.Value) interface{} {
        if len(args) > 0 {
            code := args[0].String()
            // Execute the code (simplified example)
            fmt.Printf("Running code: %s\n", code)
        }
        return nil
    }))
    
    <-c
}
```

### Compile to WASM

```bash
# From the project root
GOOS=js GOARCH=wasm go build -o assets/wasm/go.wasm cmd/wasm/main.go
```

## Environment Variables

- `GOOS=js` - Set the operating system to JavaScript
- `GOARCH=wasm` - Set the architecture to WebAssembly

## Output

The compiled binary will be written to `assets/wasm/go.wasm`.

## Testing

1. Serve the files with a web server:
   ```bash
   python -m http.server 8080
   # or
   go run github.com/caddy/caddy/v2/cmd/caddy@latest file-server --browse
   ```

2. Open your browser to `http://localhost:8080`

3. Check browser console for any errors

## Troubleshooting

### go.wasm file is too large

Try building with optimizations:

```bash
GOOS=js GOARCH=wasm go build -ldflags="-s -w" -o assets/wasm/go.wasm cmd/wasm/main.go
```

This strips debug symbols and reduces file size.

### WebAssembly not loading

1. Check that the file exists: `ls -la assets/wasm/go.wasm`
2. Verify the MIME type is set to `application/wasm`
3. Check browser console for CORS errors
